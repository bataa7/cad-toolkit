## 演示步骤

### 第一部分：测试API

#### 1. 在浏览器中访问
打开浏览器，访问以下地址：

**主页:**
```
http://localhost:5000
```
应该看到一个漂亮的欢迎页面，列出所有可用的API端点。

**通知API:**
```
http://localhost:5000/api/notifications
```
应该看到JSON格式的通知列表，包含5条测试通知。

**更新API:**
```
http://localhost:5000/api/check_update
```
应该看到JSON格式的更新信息，版本号为1.1.0。

#### 2. 使用命令行测试
```bash
# 测试通知API
python -c "import requests; r = requests.get('http://localhost:5000/api/notifications'); print('通知数量:', len(r.json()['notifications']))"

# 测试更新API
python -c "import requests; r = requests.get('http://localhost:5000/api/check_update'); print('最新版本:', r.json()['version'])"
```

### 第二部分：运行测试程序

#### 1. 启动测试程序
```bash
python simple_test.py
```

程序窗口应该显示：
- 标题: "消息推送和更新系统测试 v1.0.0"
- 日志区域显示初始化信息
- 4个测试按钮

#### 2. 测试API连接
点击 **"1. 测试API连接"** 按钮

日志应该显示：
```
🔍 测试API连接...
✅ 通知API正常 - 获取到 5 条通知
✅ 更新API正常 - 最新版本: 1.1.0
```

#### 3. 获取通知
点击 **"2. 获取通知"** 按钮

日志应该显示：
```
📥 正在获取通知...
✅ 获取成功 - 总计: 5 条, 未读: 5 条
  🔴 紧急安全更新
  ⚠️ 系统维护通知
  ✅ 新功能上线：增量更新
  ... 还有 2 条通知
```

#### 4. 显示通知中心
点击 **"3. 显示通知中心"** 按钮

应该弹出通知中心对话框，包含：
- 左侧：通知列表（5条通知，未读的显示为粗体）
- 右侧：通知详情（HTML格式，带颜色和样式）
- 顶部：标题和"全部标记为已读"按钮
- 底部：刷新和关闭按钮

**操作演示：**
1. 点击不同的通知，右侧显示详细内容
2. 点击的通知会自动标记为已读（粗体变正常）
3. 点击"全部标记为已读"，所有通知变为已读
4. 点击"刷新"重新加载通知
5. 点击"关闭"退出对话框

#### 5. 检查更新
点击 **"4. 检查更新"** 按钮

日志应该显示：
```
🔄 正在检查更新...
📦 当前版本: 1.0.0
📦 最新版本: 1.1.0
✨ 发现新版本！
```

应该弹出更新对话框，包含：
- 标题：🎉 发现新版本！
- 版本对比：1.0.0 → 1.1.0
- 更新内容：详细的changelog
- 更新信息：增量更新 | 下载大小: 5.00 MB
- 三个按钮：立即更新、稍后提醒、跳过此版本

**操作演示：**
1. 查看更新内容
2. 点击"立即更新"会开始下载（演示环境会失败，因为没有真实的更新文件）
3. 点击"稍后提醒"关闭对话框
4. 点击"跳过此版本"会记住跳过的版本

### 第三部分：查看服务器日志

在运行测试服务器的终端窗口中，应该看到类似的请求日志：
```
127.0.0.1 - - [05/Mar/2026 14:16:24] "GET /api/notifications HTTP/1.1" 200 -
127.0.0.1 - - [05/Mar/2026 14:16:30] "GET /api/notifications HTTP/1.1" 200 -
127.0.0.1 - - [05/Mar/2026 14:16:35] "GET /api/check_update HTTP/1.1" 200 -
```

每个请求都显示：
- IP地址
- 时间戳
- 请求方法和路径
- 状态码（200表示成功）

## 常见问题演示

### Q1: 服务器未启动
**现象:** 点击按钮后显示连接错误

**解决:** 
```bash
python test_server.py
```

### Q2: 端口被占用
**现象:** 服务器启动失败，提示端口5000被占用

**解决:** 
1. 关闭占用端口的程序
2. 或修改 `test_server.py` 中的端口号

### Q3: 通知无法获取
**现象:** 点击获取通知后显示错误

**检查:**
1. 服务器是否运行
2. API地址是否正确
3. 网络连接是否正常
