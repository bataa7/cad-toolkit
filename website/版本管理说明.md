# CAD工具包 - 版本管理系统说明

## 系统概述

这是一个完整的软件版本管理和自动更新系统，包括：

1. **Web端版本展示** - 在网站上展示最新版本和更新历史
2. **版本管理工具** - 用于发布新版本和管理版本信息
3. **客户端更新检查** - 可集成到CAD工具包中，自动检查更新

## 文件说明

### Web端文件

- `version.json` - 版本信息数据文件（核心）
- `version-check.js` - 前端版本检查和展示脚本
- `index.html` - 网站主页（已集成版本展示）
- `style.css` - 样式文件（包含版本相关样式）

### 管理工具

- `update-version.py` - 版本发布管理工具
- `version_checker.py` - 客户端版本检查模块

## 使用指南

### 一、发布新版本

#### 方法1：使用管理工具（推荐）

1. 进入website目录
2. 运行版本管理工具：
   ```bash
   python update-version.py
   ```

3. 选择"1. 发布新版本"

4. 按提示输入：
   - 新版本号（如：3.1.0）
   - 版本类型（重大更新/功能更新/修复更新）
   - 更新内容（每行一条）

5. 工具会自动：
   - 更新version.json
   - 计算文件大小
   - 生成发布日期
   - 插入更新历史

#### 方法2：手动编辑

直接编辑 `version.json` 文件：

```json
{
  "latest_version": "3.1.0",
  "release_date": "2026-03-10",
  "download_url": "CAD工具包.exe",
  "file_size": "52428800",
  "changelog": [
    {
      "version": "3.1.0",
      "date": "2026-03-10",
      "type": "minor",
      "changes": [
        "新增功能1",
        "优化功能2",
        "修复bug3"
      ],
      "download_count": 0
    }
  ]
}
```

### 二、部署更新

#### 本地部署

1. 确保新版本的exe文件已复制到website目录
2. 启动本地服务器：
   ```bash
   cd website
   python -m http.server 8000
   ```
3. 访问 http://localhost:8000

#### 在线部署

1. 将website目录内容上传到服务器
2. 确保version.json可以通过HTTP访问
3. 更新客户端的update_url配置

### 三、集成到CAD工具包

#### 步骤1：添加版本检查模块

将 `version_checker.py` 复制到CAD工具包项目目录

#### 步骤2：在主程序中集成

在 `cad_toolkit_gui.py` 中添加：

```python
from version_checker import VersionChecker, show_update_dialog

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        # ... 其他初始化代码
        
        # 检查更新
        self.check_for_updates()
    
    def check_for_updates(self):
        """检查更新"""
        CURRENT_VERSION = "3.0.0"  # 当前版本
        UPDATE_URL = "http://localhost:8000/version.json"  # 更新服务器
        
        checker = VersionChecker(CURRENT_VERSION, UPDATE_URL)
        update_info = checker.check_for_updates()
        
        if update_info and update_info['has_update']:
            show_update_dialog(update_info)
```

#### 步骤3：添加菜单项

```python
# 在菜单栏添加"检查更新"选项
help_menu = self.menuBar().addMenu('帮助')
check_update_action = QAction('检查更新', self)
check_update_action.triggered.connect(self.check_for_updates)
help_menu.addAction(check_update_action)
```

### 四、Web端功能

#### 自动加载版本信息

网站会自动从 `version.json` 加载：
- 最新版本号
- 发布日期
- 文件大小
- 下载次数
- 更新历史

#### 更新日志展示

用户点击"查看更新日志"按钮可以看到：
- 所有历史版本
- 每个版本的更新内容
- 版本类型标签（重大更新/功能更新/修复更新）
- 下载统计

#### 更新通知

如果用户之前下载过旧版本，再次访问网站时会：
- 自动检测本地版本（通过localStorage）
- 显示更新通知弹窗
- 引导用户下载新版本

### 五、版本号规范

采用语义化版本号（Semantic Versioning）：

```
主版本号.次版本号.修订号

例如：3.1.2
```

- **主版本号（Major）**：重大更新，可能不兼容旧版本
  - 示例：1.0.0 → 2.0.0
  - 类型：major

- **次版本号（Minor）**：功能更新，向下兼容
  - 示例：2.0.0 → 2.1.0
  - 类型：minor

- **修订号（Patch）**：bug修复，向下兼容
  - 示例：2.1.0 → 2.1.1
  - 类型：patch

### 七、测试更新功能

#### 测试Web端

1. 启动本地服务器
2. 在浏览器中打开 http://localhost:8000
3. 检查版本信息是否正确显示
4. 点击"查看更新日志"测试展开功能

#### 测试客户端检查

1. 运行测试脚本：
   ```bash
   python version_checker.py
   ```

2. 模拟旧版本：
   ```python
   CURRENT_VERSION = "2.5.0"  # 改为旧版本
   ```

3. 应该会显示更新提示

#### 测试更新通知

1. 在浏览器控制台执行：
   ```javascript
   localStorage.setItem('installed_version', '2.5.0');
   location.reload();
   ```

2. 应该会弹出更新通知

### 八、高级配置

#### 自定义更新服务器

修改客户端的update_url：

```python
# 本地测试
UPDATE_URL = "http://localhost:8000/version.json"

# 生产环境
UPDATE_URL = "https://your-domain.com/version.json"
```

#### 禁用自动检查

在客户端添加配置选项：

```python
class Settings:
    AUTO_CHECK_UPDATE = True  # 是否自动检查更新
    CHECK_INTERVAL = 7  # 检查间隔（天）
```

#### 强制更新

在version.json中添加：

```json
{
  "minimum_version": "2.8.0",
  "force_update": true,
  "force_update_message": "此版本存在严重bug，请立即更新"
}
```

## 故障排除

### 问题1：网站无法加载版本信息

- 检查version.json文件是否存在
- 检查JSON格式是否正确
- 查看浏览器控制台错误信息

### 问题2：客户端无法检查更新

- 检查网络连接
- 确认update_url是否正确
- 检查服务器是否运行

### 问题3：更新通知不显示

- 清除浏览器localStorage
- 检查版本号比较逻辑
- 查看JavaScript控制台错误
