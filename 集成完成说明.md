## 🔧 配置说明

### 修改API地址

编辑 `system_config.py`：

```python
# 消息推送配置
NOTIFICATION_CONFIG = {
    "api_url": "http://localhost:5000/api/notifications",  # 修改为你的服务器地址
    "enabled": True,  # 设为False可禁用通知功能
}

# 更新系统配置
UPDATE_CONFIG = {
    "api_url": "http://localhost:5000/api/check_update",  # 修改为你的服务器地址
    "enabled": True,  # 设为False可禁用更新功能
    "check_on_startup": True,  # 启动时检查更新
}
```

### 禁用某个功能

```python
# 只禁用通知
NOTIFICATION_CONFIG['enabled'] = False

# 只禁用更新
UPDATE_CONFIG['enabled'] = False

# 禁用启动时检查更新
UPDATE_CONFIG['check_on_startup'] = False
```

### 修改检查频率

```python
# 通知检查间隔（秒）
NOTIFICATION_CONFIG['check_interval'] = 600  # 改为10分钟

# 更新检查间隔（秒）
UPDATE_CONFIG['auto_check_interval'] = 7200  # 改为2小时
```

## 🐛 故障排除

### 问题1：状态栏没有通知按钮

**可能原因：**
- 通知系统未启用
- 导入失败

**解决方法：**
1. 检查 `system_config.py` 中 `NOTIFICATION_CONFIG['enabled']` 是否为 `True`
2. 检查是否有导入错误（查看控制台输出）
3. 确认所有文件都在同一目录

### 问题2：点击通知按钮没反应

**可能原因：**
- 服务器未启动
- API地址配置错误

**解决方法：**
1. 启动测试服务器：`python test_server.py`
2. 检查 `system_config.py` 中的API地址
3. 测试API是否可访问：在浏览器打开 http://localhost:5000/api/notifications

### 问题3：检查更新失败

**可能原因：**
- 服务器未启动
- 网络连接问题

**解决方法：**
1. 确认测试服务器正在运行
2. 检查防火墙设置
3. 查看错误提示信息

### 问题4：程序启动报错

**可能原因：**
- 缺少依赖库

**解决方法：**
```bash
pip install requests flask
```
