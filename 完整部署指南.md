## 🚀 部署步骤

### 第一步: 创建GitHub仓库

#### 1.1 访问GitHub创建页面

打开浏览器，访问: https://github.com/new

#### 1.2 填写仓库信息

```
Repository name: cad-toolkit
Description: CAD文件处理工具 - 支持块导出、文件读取、自动更新
Public/Private: 选择 Public（推荐）

⚠️ 重要：不要勾选以下选项
[ ] Add a README file
[ ] Add .gitignore
[ ] Choose a license
```

#### 1.3 创建仓库

点击绿色按钮 "Create repository"

### 第二步: 推送代码到GitHub

#### 2.1 准备认证

GitHub现在需要Personal Access Token（PAT）来推送代码：

1. 访问: https://github.com/settings/tokens
2. 点击 "Generate new token" -> "Generate new token (classic)"
3. 填写信息:
   - **Note**: `CAD Toolkit Upload`
   - **Expiration**: 选择 `90 days` 或更长
   - **Select scopes**: 勾选 `repo`（完整仓库访问权限）
4. 点击 "Generate token"
5. **立即复制token**（只显示一次！保存到安全的地方）

#### 2.2 推送代码

**方法1: 使用脚本（推荐）**

双击运行 `推送到GitHub.bat`

**方法2: 手动命令**

打开命令行，执行：

```bash
git push -u origin main
```

当提示输入凭据时：
- Username: `bataa7`
- Password: `粘贴你的Personal Access Token`

#### 2.3 验证推送

访问: https://github.com/bataa7/cad-toolkit

应该能看到所有文件，包括：
- README.md
- update_manager.py
- version.json
- docs/ 文件夹
- 等等...

### 第三步: 创建第一个Release

### 📥 安装说明

4. 点击 "Publish release"

### 第四步: 部署到Netlify

#### 4.1 登录Netlify

1. 访问: https://www.netlify.com/
2. 点击 "Sign up" 或 "Log in"
3. 选择 "GitHub" 登录（推荐）
4. 授权Netlify访问你的GitHub账户

#### 4.2 创建新站点

1. 点击 "Add new site" -> "Import an existing project"
2. 选择 "GitHub"
3. 在仓库列表中找到并选择 `bataa7/cad-toolkit`
4. 如果看不到仓库，点击 "Configure the Netlify app on GitHub" 添加访问权限

#### 4.3 配置构建设置

Netlify会自动检测 `netlify.toml` 配置，确认以下设置：

```
Branch to deploy: main
Build command: echo 'Static site - no build needed'
Publish directory: docs
```

点击 "Deploy site"

#### 4.4 等待部署完成

- 部署通常需要1-2分钟
- 完成后会显示一个临时域名，如: `random-name-123456.netlify.app`
- 点击域名访问你的网站

#### 4.5 自定义站点名称（可选）

1. 在站点设置中点击 "Site settings"
2. 点击 "Change site name"
3. 输入新名称，如: `cad-toolkit`
4. 保存后，你的网站地址变为: `https://cad-toolkit.netlify.app`

## 🔧 故障排除

### 问题1: 推送到GitHub失败

**错误**: `Authentication failed`

**解决方法**:
1. 确认使用的是Personal Access Token，不是GitHub密码
2. 检查Token是否有 `repo` 权限
3. Token可能已过期，需要重新生成

### 问题2: Netlify部署失败

**错误**: `Build failed`

**解决方法**:
1. 检查 `netlify.toml` 配置是否正确
2. 确认 `docs` 目录存在且包含 `index.html`
3. 查看部署日志获取详细错误信息

### 问题3: 更新检查失败

**错误**: `Failed to fetch version info`

**解决方法**:
1. 确认GitHub仓库是Public
2. 检查 `version.json` 文件是否存在
3. 确认网络连接正常
4. 检查GitHub API是否可访问

### 问题4: 网站无法访问

**错误**: `Site not found`

**解决方法**:
1. 等待几分钟让DNS传播
2. 清除浏览器缓存
3. 检查Netlify部署状态
4. 确认站点没有被暂停
