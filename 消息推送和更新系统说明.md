# 消息推送和增量更新系统说明

## 概述

本系统为CAD工具包添加了两个重要功能：
1. **消息推送系统** - 从服务器获取并显示通知、公告等信息
2. **增量更新系统** - 支持差分更新，只下载变化的文件，节省流量和时间

## 文件结构

```
notification_system.py      # 消息推送系统核心模块
update_system.py           # 增量更新系统核心模块
system_config.py           # 系统配置文件
integration_example.py     # 集成示例代码
test_notifications.json    # 测试用通知数据
test_update_info.json      # 测试用更新信息
```

## 功能特性

### 消息推送系统

- ✅ 从服务器获取通知列表
- ✅ 支持多种通知级别（信息、警告、严重、成功）
- ✅ 未读消息标记和徽章显示
- ✅ 通知中心界面，查看所有历史通知
- ✅ 本地缓存已读状态
- ✅ 定时自动检查新通知
- ✅ 支持HTML格式的通知内容

### 增量更新系统

- ✅ 版本检查和比较
- ✅ 增量更新（只下载变化的文件）
- ✅ 完整包更新（备用方案）
- ✅ 文件哈希校验，确保完整性
- ✅ 自动备份和恢复机制
- ✅ 更新进度显示
- ✅ 支持跳过特定版本
- ✅ 更新后自动重启

## 快速开始

### 1. 配置服务器API

编辑 `system_config.py`：

```python
# 消息推送API
NOTIFICATION_CONFIG = {
    "api_url": "https://your-server.com/api/notifications",
    "enabled": True,
}

# 更新检查API
UPDATE_CONFIG = {
    "api_url": "https://your-server.com/api/check_update",
    "enabled": True,
}
```

### 2. 集成到主程序

参考 `integration_example.py`，在主窗口中添加：

```python
from notification_system import NotificationManager, NotificationWidget
from update_system import UpdateChecker, UpdateDialog
from system_config import NOTIFICATION_CONFIG, UPDATE_CONFIG, APP_VERSION

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        
        # 初始化通知管理器
        self.notification_manager = NotificationManager(
            api_url=NOTIFICATION_CONFIG['api_url']
        )
        
        # 添加通知小部件到工具栏
        self.notification_widget = NotificationWidget(
            self.notification_manager, self
        )
        toolbar.addWidget(self.notification_widget)
        
        # 初始化更新系统
        self.init_update_system()
```

### 3. 测试功能

使用本地测试数据：

```python
# 在 system_config.py 中启用开发模式
DEV_MODE = {
    "use_local_data": True,
    "local_notifications": "test_notifications.json",
    "local_update_info": "test_update_info.json",
}
```

运行示例程序：
```bash
python integration_example.py
```

## API接口规范

### 通知API

**请求：** `GET https://your-server.com/api/notifications`

**响应格式：**
```json
{
  "notifications": [
    {
      "id": "notif_001",
      "title": "通知标题",
      "content": "通知内容（支持HTML）",
      "level": "info",  // info, warning, critical, success
      "date": "2026-03-05"
    }
  ]
}
```

### 更新检查API

**请求：** `GET https://your-server.com/api/check_update`

**响应格式：**
```json
{
  "version": "1.1.0",
  "changelog": "更新说明",
  "size": 5242880,
  "release_date": "2026-03-05",
  "patch_files": [
    {
      "name": "patch_001.json",
      "url": "https://example.com/updates/patch_001.json",
      "hash": "sha256_hash",
      "target": "target_file.py",
      "action": "replace",  // replace, create, delete
      "source": "source_file.py"
    }
  ],
  "full_package": {
    "name": "package.zip",
    "url": "https://example.com/updates/package.zip",
    "hash": "sha256_hash"
  }
}
```

## 增量更新原理

### 补丁文件格式

每个补丁文件是一个JSON，描述如何更新目标文件：

```json
{
  "target": "cad_toolkit_gui.py",
  "action": "replace",
  "source": "cad_toolkit_gui.py",
  "hash": "file_hash"
}
```

### 更新流程

1. **检查版本** - 比较本地版本和服务器最新版本
2. **下载补丁** - 下载所有需要的补丁文件
3. **验证哈希** - 确保文件完整性
4. **备份原文件** - 自动备份将被修改的文件
5. **应用补丁** - 根据补丁描述更新文件
6. **验证更新** - 检查更新是否成功
7. **重启程序** - 应用新版本

如果更新失败，系统会自动恢复备份。

## 服务器端实现示例

### 使用Flask实现API

```python
from flask import Flask, jsonify
import json

app = Flask(__name__)

@app.route('/api/notifications')
def get_notifications():
    with open('notifications.json', 'r', encoding='utf-8') as f:
        data = json.load(f)
    return jsonify(data)

@app.route('/api/check_update')
def check_update():
    with open('update_info.json', 'r', encoding='utf-8') as f:
        data = json.load(f)
    return jsonify(data)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
```

### 使用静态文件服务

如果不想搭建动态服务器，可以使用静态文件托管：

1. 将 `notifications.json` 和 `update_info.json` 上传到服务器
2. 配置API地址为文件的直接URL
3. 定期更新这些JSON文件即可

例如使用GitHub Pages：
```python
NOTIFICATION_CONFIG = {
    "api_url": "https://username.github.io/cad-toolkit/notifications.json",
}
```

## 安全建议

1. **使用HTTPS** - 确保API使用HTTPS协议
2. **文件哈希验证** - 始终验证下载文件的哈希值
3. **签名验证** - 可以添加数字签名验证更新包的真实性
4. **备份机制** - 更新前自动备份，失败时恢复
5. **版本回退** - 保留历史版本，支持回退

### 自定义更新策略

```python
class CustomUpdateChecker(UpdateChecker):
    def should_update(self, update_info):
        # 自定义更新策略
        if update_info.get('force_update'):
            return True
        # 只在WiFi环境下自动更新
        return self.is_wifi_connected()
```

### 更新回调

```python

downloader = UpdateDownloader(update_info, download_dir)
downloader.progress_updated.connect(on_update_progress)
```

## 故障排除

### 通知无法获取

1. 检查网络连接
2. 验证API地址是否正确
3. 查看日志文件中的错误信息
4. 测试API是否可以在浏览器中访问

### 更新失败

1. 检查磁盘空间是否充足
2. 确认有文件写入权限
3. 查看备份文件是否存在
4. 手动下载完整包进行更新

### 更新后程序无法启动

1. 从备份目录恢复文件
2. 重新下载完整安装包
3. 检查日志文件中的错误信息
