这些是警告，不是错误，不影响程序功能。

## 已修复的问题

### 1. 线程管理问题
**问题：** 线程对象没有保存为实例变量，导致被垃圾回收

**修复：**
```python
# 修复前
def fetch_notifications(self):
    fetcher = NotificationFetcher(...)  # 局部变量
    fetcher.start()

# 修复后
def fetch_notifications(self):
    self.notification_fetcher = NotificationFetcher(...)  # 实例变量
    self.notification_fetcher.start()
```

### 2. 启动时自动检查更新
**问题：** 启动时立即检查更新可能导致网络延迟

**修复：** 默认禁用启动时检查更新
```python
UPDATE_CONFIG = {
    "check_on_startup": False,  # 改为False
}
```

## 关于警告信息

### QLayout::addChildLayout 警告
这是原有代码中的布局问题，不是新增代码导致的。不影响功能，可以忽略。

### QThread 警告
这是因为程序退出时线程还在运行。已经通过保存线程引用来改善，但在快速退出时仍可能出现。不影响正常使用。

## 配置说明

### 启用/禁用功能

编辑 `system_config.py`：

```python
# 禁用通知系统
NOTIFICATION_CONFIG['enabled'] = False

# 禁用更新系统
UPDATE_CONFIG['enabled'] = False

# 启用启动时检查更新
UPDATE_CONFIG['check_on_startup'] = True
```

### 修改API地址

```python
# 使用本地测试服务器
NOTIFICATION_CONFIG['api_url'] = "http://localhost:5000/api/notifications"
UPDATE_CONFIG['api_url'] = "http://localhost:5000/api/check_update"

# 使用生产服务器
NOTIFICATION_CONFIG['api_url'] = "https://your-server.com/api/notifications"
UPDATE_CONFIG['api_url'] = "https://your-server.com/api/check_update"
```
